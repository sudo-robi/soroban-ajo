## ğŸ¯ Key Design Decisions

### 1. Polling vs WebSocket
**Decision**: Polling (30s interval)
**Reason**: 
- Simpler implementation
- Better browser compatibility
- Sufficient for non-critical updates
- Easier to debug

### 2. Service Layer Pattern
**Decision**: Separate service classes
**Reason**:
- Clear separation of concerns
- Easier testing
- Reusable across components
- Centralized error handling

### 3. Mock Data Fallback
**Decision**: Automatic fallback to mock data
**Reason**:
- Development without blockchain
- Graceful degradation
- UI testing without contracts
- Demo purposes

### 4. TypeScript Full Coverage
**Decision**: Strict TypeScript
**Reason**:
- Type safety
- Better IDE support
- Fewer runtime errors
- Self-documenting code

### 5. Component Composition
**Decision**: Small, focused components
**Reason**:
- Reusability
- Easier testing
- Better performance (React.memo)
- Cleaner code
